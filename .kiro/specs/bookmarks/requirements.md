# ブックマークサービス 要件定義書

## 概要

個人利用のブックマークサービス。URLをPOSTすると、生成AIがWebページの内容を要約し、URLと要約を保存する。保存したブックマークはあとから一覧表示・検索して見返すことができる。

## 技術スタック

- バックエンド: Ruby 4.0 / Rails 8.1（APIモード）
- データベース: SQLite3（開発/テスト）、PostgreSQL 16（本番）
- AI要約: Amazon Bedrock（Claude）
- 認証: APIキーによる簡易認証

---

## 1. ユーザーストーリー

### 1.1 ブックマークの登録

**ユーザーとして**、URLをPOSTすることで、Webページの要約を自動生成し、ブックマークとして保存したい。**それにより**、あとから内容を思い出しやすくなる。

#### 受け入れ基準

- URLをPOSTリクエストで送信できる
- システムがURLのWebページを取得し、内容を解析する
- Amazon Bedrock（Claude）を使用してページ内容の要約を生成する
- URL、要約、ページタイトル、登録日時がデータベースに保存される
- 登録完了後、保存されたブックマーク情報がレスポンスとして返却される
- 無効なURLや取得できないページの場合、適切なエラーメッセージが返される

### 1.2 ブックマークの一覧表示

**ユーザーとして**、保存したブックマークを一覧で確認したい。**それにより**、過去に保存したページを見返すことができる。

#### 受け入れ基準

- GET リクエストで保存済みブックマークの一覧を取得できる
- 一覧には URL、タイトル、要約、登録日時が含まれる
- 新しい順（登録日時の降順）で表示される
- ページネーションに対応している（デフォルト20件/ページ）

### 1.3 ブックマークの検索

**ユーザーとして**、キーワードでブックマークを検索したい。**それにより**、目的のブックマークを素早く見つけることができる。

#### 受け入れ基準

- キーワードを指定して保存済みブックマークを検索できる
- タイトルと要約の両方が検索対象となる
- 検索結果はページネーションに対応している
- 部分一致で検索できる

### 1.4 ブックマークの削除

**ユーザーとして**、不要になったブックマークを削除したい。**それにより**、一覧を整理できる。

#### 受け入れ基準

- DELETEリクエストで指定したブックマークを削除できる
- 存在しないブックマークを削除しようとした場合、404エラーが返される
- 削除は物理削除とする

### 1.5 ブックマークの詳細表示

**ユーザーとして**、特定のブックマークの詳細を確認したい。**それにより**、要約の全文やメタ情報を確認できる。

#### 受け入れ基準

- GET リクエストでブックマークIDを指定して詳細情報を取得できる
- URL、タイトル、要約全文、登録日時、最終更新日時が含まれる
- 存在しないIDの場合、404エラーが返される

## 2. 機能要件

### 2.1 API エンドポイント

| メソッド | パス | 説明 |
|---------|------|------|
| POST | /api/v1/bookmarks | ブックマークの登録 |
| GET | /api/v1/bookmarks | ブックマーク一覧の取得 |
| GET | /api/v1/bookmarks/:id | ブックマーク詳細の取得 |
| DELETE | /api/v1/bookmarks/:id | ブックマークの削除 |
| GET | /api/v1/bookmarks/search | ブックマークの検索 |

### 2.2 ブックマーク登録処理

1. クライアントがURLを含むPOSTリクエストを送信する
2. URLの形式バリデーションを行う
3. URLのWebページをHTTPで取得する
4. HTMLからページタイトルと本文テキストを抽出する
5. 抽出したテキストをAmazon Bedrock（Claude）に送信し、要約を生成する
6. URL、タイトル、要約、メタデータをデータベースに保存する
7. 保存結果をJSON形式でレスポンスとして返却する

### 2.3 データモデル

#### bookmarks テーブル

| カラム名 | 型 | 説明 |
|---------|------|------|
| id | bigint (PK) | 一意識別子 |
| url | text | ブックマークURL（一意制約） |
| title | text | ページタイトル |
| summary | text | AI生成の要約 |
| created_at | timestamp | 登録日時 |
| updated_at | timestamp | 更新日時 |

### 2.4 AI要約機能

- Amazon Bedrock の Claude モデルを使用する
- 要約は日本語で生成する（ページが英語の場合も日本語で要約）
- 要約の長さは200〜400文字程度を目安とする
- ページ内容が取得できない場合は、要約なしで登録し、エラーステータスを記録する

### 2.5 Webページ取得

- HTTP/HTTPS のURLに対応する
- リダイレクトに追従する（最大5回）
- タイムアウトは10秒とする
- User-Agentヘッダーを適切に設定する
- HTMLのメタタグ（og:title、og:description）も取得対象とする

### 2.6 認証

- APIキーによる簡易認証を採用する
- リクエストヘッダー `X-API-Key` でAPIキーを送信する
- APIキーは環境変数で管理する
- APIキーが不正な場合、401 Unauthorized を返す

### 2.7 重複チェック

- 同じURLが既に登録されている場合、409 Conflict を返す
- 重複判定はURL正規化後に行う（末尾スラッシュの統一など）

## 3. 非機能要件

### 3.1 パフォーマンス

- ブックマーク一覧・検索のレスポンスタイムは500ms以内を目標とする
- ブックマーク登録はWebページ取得とAI要約を含むため、30秒以内を目標とする
- データベースの適切なインデックスを設定する（url, created_at）

### 3.2 信頼性

- Webページ取得失敗時もブックマーク自体は登録可能とする（タイトル・要約は空）
- AI要約生成失敗時もブックマーク自体は登録可能とする（要約は空）
- エラー発生時は適切なHTTPステータスコードとエラーメッセージを返す

### 3.3 セキュリティ

- APIキーによるアクセス制御を実装する
- 入力URLのバリデーション（形式チェック、プライベートIPへのアクセス防止）
- SQLインジェクション対策（ActiveRecordのパラメータバインディング使用）
- レートリミット：1分あたり10リクエストまで

### 3.4 運用性

- ログ出力：リクエスト/レスポンス、エラー、AI API呼び出しのログを記録する
- ヘルスチェックエンドポイント（GET /health）を提供する
- 環境変数で設定を外部化する（DB接続情報、APIキー、Bedrock設定）

### 3.5 開発環境

- 開発環境はローカルで簡単に起動できること（SQLite3使用のためDB構築不要）
- Minitest によるテストを整備すること
- RuboCop によるコードスタイルチェックを導入すること

### 3.6 制約事項

- 個人利用のため、マルチユーザー対応は不要
- フロントエンド（UI）は本スコープに含まない（APIのみ提供）
- ファイルアップロードやスクリーンショット取得は対象外
- ブックマークの編集（更新）機能は初期スコープでは対象外
